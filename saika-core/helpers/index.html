Helpers

Helpers are the files that contains custom user defined functions. Off course you can define your custom functions in other files and include them in your code. But Saika's helpers allows you to autoload them or loading them on demand. You have to place your controllers in app/helpers/ directory.

Sample Helper Structure

There is no strict structure for helper files. However, we recommend that you follow PSR standards.

A demo helper could be like this:

app/helpers/DemoHelper.php

<?php

/**
 * Write your custom functions here
 */

function howdy()
{
    echo "Howdy Watson!";
}

Tip

It is recommended that you use function_exists() while declaring functions to avoid any conflicts.

Loading Helpers

You can load a helper any where in your code by using the function load_helper(). The function takes a helper file name without the .php extension as it's only argument. A sample usage:

<?php
class IndexController extends Controller
{
    /**
     * Construct this object by extending the base Controller class
     */
    public function __construct()
    {
        parent::__construct();
        // The file app/helpers/DemoHelper.php will be loaded
        load_helper('DemoHelper');
    }

However, it's also possible to autoload some ( or all of 'em ) helpers when Saika boots up. To do this you have to set the configuration value $_config['AUTOLOAD_HELPERS'] with name of Helper files without extension (.php) as array. Ex. ['DemoHelper', 'UrlHelpers']
